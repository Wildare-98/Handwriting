# 门诊病例手写渲染技能（real-handwriting）

从用户提供的病例图片中提取信息，按标准模板生成规范的门诊病历 .txt 文档，
调用 Handwritingv1.1 加密工具渲染手写版，并同步维护病例登记表（含手写文件路径）。

## 触发条件

- 用户提供病例图片（或截图），要求提取/录入/生成门诊病例。
- 用户提及"门诊病例""病例提取""生成病历""手写病例"等关键词。
- 用户说"按模板处理这批病例图片"。

## 工作流程

### 0. 环境与依赖确认（必须首先执行）

在处理任何病例之前，必须确认运行环境满足以下条件：

#### 0.1 Python 版本检查

Handwritingv1.1 为 Pyarmor 加密版本，其 `.pyd` 解密运行时仅绑定了 **Python 3.13 (64-bit, Windows)**。
**其他 Python 版本将无法加载解密模块，导致渲染失败。**

验证方式：
```bash
python --version
```

预期输出包含 `Python 3.13`。若系统默认 Python 不是 3.13，则需使用显式的 Python 3.13 路径（如 `python3.13`、`py -3.13`，或在各步骤中替换为实际的 3.13 可执行文件路径）。

> **关键约束**：以下所有调用 Handwritingv1.1 的命令必须使用 Python 3.13 执行。
> 其他脚本（HEIC 转换、PDF 合并）可使用任意 Python 版本。

#### 0.2 依赖安装验证

以下依赖必须已安装。若缺失，自动安装：

| 依赖 | 用途 |
|------|------|
| `Pillow` | 图片读写、手写体渲染 |
| `reportlab` | PDF 生成 |
| `numpy` | 高斯噪声、晕影效果（可选） |
| `pillow_heif` | HEIC 格式转换 |
| `pypdf` | PDF 合并 |

一键安装：
```bash
pip install Pillow reportlab numpy pillow_heif pypdf
```

#### 0.3 字体文件确认

Handwritingv1.1 **不含字体文件**。执行前确认用户已提供手写字体（TTF/OTF/TTC），并记下字体文件的路径 `{FONT_PATH}`。

#### 0.4 License 状态确认

执行 `--license-info` 检查 License 余量，确保配额充足（每 1000 字符消耗 1 次调用）：
```bash
python {SKILL_DIR}/tools/Handwritingv1.1/handwriting.py --license-info
```

若未设置 License，提示用户执行 `--license-setup <key>`。

若以上任一检查未通过，**暂停并向用户报告具体问题，等待解决后再继续**。

---

### 1. HEIC 格式预处理（按需）

用户提供的病例图片若为 HEIC 格式，需先转为 JPG。技能自带转换脚本 `tools/convert_heic_to_jpg.py`。

转换方式：编写临时代码，遍历图片目录中的 `.HEIC` 文件，用 `pillow_heif` + `PIL` 转为 JPG 并保存到同一目录（或 `_jpg` 子目录），质量设为 40（病例文字可辨即可）。

转换完成后，后续所有步骤使用 JPG 文件。

### 输出目录约定

所有生成的文件统一放在一个独立的输出目录下，按类型分文件夹存放：

```
{来源科室}_输出/              # 如"心内科病例_输出"，混合科室时用"病例输出"
├── txt_files/                # 所有 .txt 病例文档
├── handwriting/              # 各病例手写版子目录
│   ├── 01_门诊病例_张三_日期/
│   └── ...
├── 病例登记表.txt             # 登记表（含手写路径列）
└── 全部病例_手写版.pdf        # 合并后的手写 PDF
```

- 处理开始前创建此目录结构。
- 后续各步骤中的 `{输出目录}` 即指此根目录。

### 2. 读取病例图片并合并多图病例

读取用户提供的所有图片。

**多图合并识别**：一份完整病例可能由两张图片组成（如病例主体一图、处方或检查单独一图）。读取时按以下规则甄别：
- 检查相邻图片的**姓名**和**门诊号**是否相同 → 相同则属于同一份病例，合并信息。
- 检查图片内容是否属于同一病例的延续（如前图截断于现病史、后图从既往史开始）→ 合并。
- 合并后按一份病例处理，只生成一份 .txt、一条登记记录。

合并完成后，逐份提取所有可见的病例信息，包括但不限于：
- 基本信息：姓名、性别、年龄、就诊时间、科室、门诊号
- 主诉、现病史、既往史、过敏史、个人史、家族史
- 生命体征、体格检查、中医四诊信息
- 辅助检查、诊断、处置、注意事项

### 3. 校验就诊日期

提取到就诊日期后，获取当前系统时间进行比对：
- 若就诊日期**晚于当前日期**（如当前为 2026 年，就诊日期为 2028 年），这可能是图片 OCR 识别错误或图片日期有误。
- **必须向用户报告**：列出所有就诊日期异常的病历，提示用户核对并提供正确日期，**暂停后续处理，等待用户确认后再继续**。
- 若就诊日期正常（不晚于当前），直接进入下一步。

### 4. 按规则补全缺失字段

读取 `references/filling-rules.md`，逐项检查：
- 图片中已有的信息 → 如实使用
- 图片中缺失或模糊的信息 → 按填充规则处理
- **关键原则**：需要根据病例整体情况编写的内容（中医四诊、辨证依据、治则治法、诊断）必须与图片中的主诉和现病史逻辑自洽

### 5. 生成病例 .txt 文件

以 `assets/门诊病例模板.txt` 为排版基准：
- **保留模板的标题、缩进、分段格式不变**
- 将提取和补全的信息填入对应位置
- 文件命名为 `{序号}_门诊病例_{姓名}_{就诊日期}.txt`（序号与登记表一致，两位数补零，如 `01_门诊病例_张三_2026-01-18.txt`）
- 保存到输出目录的 `txt_files/` 子文件夹下

### 6. 更新登记表

维护输出目录下的 `病例登记表.txt`，格式如下：

```
======================================================================
                北京xxx病例登记表
======================================================================
序号 | 姓名   | 性别 | 门诊号     | 就诊日期   | 中医诊断           | 西医诊断         | 手写病例路径
----------------------------------------------------------------------
1    | 张x国 | 男   |            | 2026/6/15  | 胃痛（脾胃虚寒证）  | 慢性浅表性胃炎   | handwriting/01_门诊病例_张x国_2026-06-15/
----------------------------------------------------------------------
```

- 首次运行时创建该文件。
- **每处理完一份病例并完成手写渲染后，追加一行记录（此时手写路径已确定）。**
- 登记表记录七项：姓名、性别、门诊号、就诊日期、中医诊断、西医诊断、手写病例路径。
- **手写病例路径**：填入相对于输出目录的路径，即 `handwriting/{序号}_门诊病例_{姓名}_{就诊日期}/`。该目录包含该患者的所有手写 PNG 页面及合并 PDF。

> **注意**：登记表更新时机在步骤 7（手写渲染）完成后，而非生成 .txt 时。因为只有渲染完成后，手写病例路径才最终确定。

### 7. 生成手写版文件

生成 .txt 后，调用 **Handwritingv1.1（Pyarmor 加密版）** 将病例转换为手写版 PNG/PDF。

Handwritingv1.1 位于本技能目录 `tools/Handwritingv1.1/` 下，**不含字体文件**，需用户提供。

**必须使用 Python 3.13** 运行（加密 `.pyd` 模块仅绑定此版本）。

运行命令：
```bash
python {SKILL_DIR}/tools/Handwritingv1.1/handwriting.py \
  --input "./{输出目录}/txt_files/{序号}_门诊病例_{姓名}_{就诊日期}.txt" \
  --font "{FONT_PATH}" \
  --out "./{输出目录}/handwriting/{序号}_门诊病例_{姓名}_{就诊日期}" \
  --paper white \
  --seed 42
```

> **注意**：
> - 执行时必须确保 `python` 为 Python 3.13（参见步骤 0.1）。
> - `{SKILL_DIR}` 表示本技能根目录的绝对路径，由 AI 在执行时自动解析。
> - `{FONT_PATH}` 为步骤 0.3 确认的用户手写字体的路径。
> - License 验证已内置于渲染流程，无需额外调用。
> - **如需自定义渲染效果**（行间距、字体抖动、纸张样式、涂改模拟等），先读取 `references/handwriting-tool.md` 获取完整参数列表及预设文件用法。

输出目录 `{输出目录}/handwriting/{序号}_门诊病例_{姓名}_{就诊日期}/` 包含：
- `page-001.png`, `page-002.png`, … — 每页 PNG
- `handwriting.pdf` — 合并后的 PDF

渲染完成后，**立即执行步骤 6 更新登记表**，将手写路径写入该病例的登记记录。

#### License 错误处理

若脚本输出以 `[License]` 开头，按以下规则处理：

| 错误信息 | 处理方式 |
|----------|----------|
| `未设置 License Key` | 提示用户执行 `--license-setup <key>` |
| `无效的 License Key` | 告知用户 Key 无效，请检查 |
| `调用次数已用完` | 告知用户配额已耗尽，需购买更多 |
| `无法连接服务器` | 告知用户网络问题，稍后重试 |

### 8. 合并全部手写 PDF

全部病例处理完毕后，将本次生成的所有手写 PDF 合并为一个文件。

本技能自带合并脚本 `tools/merge_pdf.py`，依赖 `pypdf`（已在步骤 0 中安装）。

运行命令：
```bash
python {SKILL_DIR}/tools/merge_pdf.py \
  -d "./{输出目录}/handwriting" \
  -r \
  -p "handwriting" \
  -o "./{输出目录}/全部病例_手写版.pdf"
```

说明：
- `-d` 指定手写版所在目录 `{输出目录}/handwriting/`。
- `-r` 递归扫描子目录。
- `-p "handwriting"` 只匹配名为 `handwriting` 开头的 PDF。
- **原有的独立 PDF 不做任何改动**，只新生成合并文件。
